## 迭代规则

- 每次大修订新建文件，命名格式：`{文档名}-V{主版本}.{次版本}.md`
- 修订说明写入文件末尾的版本记录表
- 接口变更需同步更新 API 接口规范文档

## 使用顺序

1. 先读 `后端开发规范-V1.0.md`，确定整体架构、项目结构、编码规范和 RAG 链路设计。
2. 再读 `数据库设计详细文档-V1.0.md`，按 DDL 创建 ORM 模型和 Alembic 迁移。
3. 然后读 `API接口规范-V1.0.md`，按接口规范实现路由和 Service。
4. 开发执行时使用 `后端开发任务清单.md`，按 Phase 0 到 Phase 4 推进并验收。
