```
frontend-design/
├── README.md                          # 版本索引
├── docs/

│   ├── 前端页面原型-V0.1.md            # 核心页面布局原型

```

## 迭代规则

- 每次大修订新建文件，命名格式：`前端设计规范-V{主版本}.{次版本}.md`
- 修订说明写入文件末尾的版本记录表
- 上线后的界面截图放入 `assets/`，在文档中引用实际截图替代文字描述

## 使用顺序

1. 先读 `前端设计规范-V1.0.md`，确定整体架构、渲染策略、SEO、a11y、状态管理和监控 SDK 边界。
2. 再读 `前端组件规范-V1.0.md`，按组件规范实现按钮、表单、表格、卡片、RAG、图表等基础组件。
3. 最后读 `前端页面原型-V0.1.md`，按页面原型拆分 Nuxt 页面和开发任务。
4. 开发执行时使用 `前端开发任务清单.md`，按 Phase 0 到 Phase 4 推进并验收。
