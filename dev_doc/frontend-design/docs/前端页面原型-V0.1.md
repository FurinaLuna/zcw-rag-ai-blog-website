# 智能内容平台前端页面原型

版本：V0.1  
日期：2026-05-03  
关联文档：`前端设计规范-V1.0.md`、`前端组件规范-V1.0.md`  
目标：把核心页面从“功能列表”细化为可实现的布局草图、内容顺序和状态要求，方便后续进入 Nuxt3 页面开发。

---

# 一、页面优先级

## P0-A 内容主链路

1. 首页 `/`
2. 文章列表 `/articles`
3. 文章详情 `/articles/[slug]`
4. 后台登录 `/admin/login`
5. 文章管理 `/admin/articles`
6. 文章编辑 `/admin/articles/new`、`/admin/articles/[id]`

## P0-B 亮点闭环

1. RAG 问答页 `/ask`
2. 前端监控基础采集
3. Docker 部署可运行

## P1 展示增强

1. 运营大盘 `/admin`
2. 监控大盘 `/admin/monitor`
3. 知识库运维 `/admin/knowledge`
4. 专题页 `/topics`
5. 搜索页 `/search`

# 二、首页 `/`

渲染模式：SSG / 预渲染  
目标：第一屏同时传达内容站、智能问答、工程化指标三件事。

## 2.1 Desktop 原型

```txt
┌────────────────────────────────────────────────────────────┐
│ Header: Logo  Articles  Topics  Ask  About          Avatar │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  智能技术知识库                         ┌────────────────┐ │
│  记录前端工程化、Nuxt实践、RAG应用       │ Ask from notes │ │
│  与性能监控。                            │                │ │
│                                          │ 输入问题...     │ │
│  [Frontend] [Nuxt] [RAG] [Performance]  │ [开始提问]      │ │
│                                          └────────────────┘ │
│                                                            │
├────────────────────────────────────────────────────────────┤
│ 精选文章                                                    │
│ ┌──────────────┐ ┌──────────────┐ ┌──────────────┐          │
│ │ ArticleCard  │ │ ArticleCard  │ │ ArticleCard  │          │
│ └──────────────┘ └──────────────┘ └──────────────┘          │
├────────────────────────────────────────────────────────────┤
│ 知识专题                                                    │
│ [前端工程化] [Vue/Nuxt] [AI/RAG] [性能优化] [项目复盘]       │
├────────────────────────────────────────────────────────────┤
│ 系统能力指标                                                │
│ [文章数] [向量分片] [平均问答耗时] [Web Vitals]              │
├────────────────────────────────────────────────────────────┤
│ Footer                                                     │
└────────────────────────────────────────────────────────────┘
```

## 2.2 Mobile 原型

```txt
Header: Logo + Menu
站点标题
简介
标签
精选文章列表
迷你问答入口
知识专题
系统能力指标
Footer
```

移动端问答入口不要放在首屏右侧，避免挤压标题和文章入口。

## 2.3 数据需求

- 最近文章 6 条；
- 精选文章 3 条；
- 专题列表；
- 文章总数；
- 向量分片总数；
- 最近向量同步时间；
- 最近 Web Vitals 摘要；
- 最近 7 天错误数。

## 2.4 状态

- 文章为空：展示“还没有文章”与后台入口提示；
- 指标无数据：展示 `--`，不显示假数字；
- 问答不可用：问答入口置灰并提示“知识库同步后可用”。

# 三、文章列表 `/articles`

渲染模式：SSG / 预渲染  
目标：让用户快速浏览、筛选和进入文章。

## 3.1 Desktop 原型

```txt
Header
PageTitle: 文章
SearchBar
FilterRow: 分类 / 标签 / 排序

┌───────────────────────────────┬──────────────────────┐
│ ArticleList                   │ Sidebar              │
│ - ArticleCard                 │ - 热门标签            │
│ - ArticleCard                 │ - 专题入口            │
│ - ArticleCard                 │ - RAG 提问入口        │
└───────────────────────────────┴──────────────────────┘

Pagination
Footer
```

## 3.2 筛选规则

- 分类筛选：单选；
- 标签筛选：多选；
- 排序：最新、阅读量、更新时间；
- 搜索：跳转 `/search?q=xxx`，不在列表页做复杂搜索。

## 3.3 状态

- loading：文章卡片骨架；
- empty：展示“没有匹配文章”，附带清空筛选按钮；
- error：展示重试按钮。

---

# 四、文章详情 `/articles/[slug]`

渲染模式：SSR  
目标：项目最重要的阅读体验页面，优先打磨。

## 4.1 Desktop 原型

```txt
Header
ReadingProgress

┌───────────────────────┬────────────────────────────┬──────────────┐
│ Left spacer           │ Article                    │ TOC          │
│                       │ Title                      │ H2           │
│                       │ Summary                    │ H3           │
│                       │ Meta + Author              │ H2           │
│                       │ Tags                       │              │
│                       │ Markdown Body              │              │
│                       │ CodeBlock + Copy           │              │
│                       │ Images / Diagrams          │              │
│                       │ Ask about this article     │              │
│                       │ RelatedArticles            │              │
└───────────────────────┴────────────────────────────┴──────────────┘

Footer
```

正文最大宽度：680px。TOC 仅 Desktop 显示，Mobile 改为文章顶部折叠目录。

## 4.2 文章标题区

必须包含：

- 标题；
- 摘要；
- 发布时间；
- 更新时间；
- 阅读时间；
- 作者头像；
- 分类；
- 标签；
- “基于本文提问”按钮。

## 4.3 Markdown 正文

要求：

- H2/H3 生成目录；
- 代码块支持语言标识；
- 代码块右上角复制按钮；
- 图片固定宽高或 aspect-ratio，避免 CLS；
- 外链打开新窗口并加 `rel="noopener noreferrer"`；
- HTML 内容必须 sanitize。

## 4.4 底部区域

顺序：

1. 基于本文提问；
2. 相关文章；
3. 评论区（P2，可暂不实现）。

首版如果不做评论，底部不要展示空评论模块。

# 五、RAG 问答页 `/ask`

渲染模式：CSR  
目标：展示站内知识库问答能力和引用来源，是面试演示重点。

## 5.1 Desktop 原型

```txt
Header

┌──────────────────────────────────────────────────────┐
│ Ask from my notes                                    │
│ 只基于站内文章和知识库回答。                          │
├──────────────────────────────────────────────────────┤
│ 示例问题 chips                                       │
│ [Nuxt3 中 SSR 和 SSG 如何选择？]                      │
│ [这个项目如何实现前端监控？]                          │
│ [pgvector 在 RAG 里负责什么？]                        │
├──────────────────────────────────────────────────────┤
│ Conversation                                         │
│ User message                                         │
│ Assistant answer streaming...                        │
│ Source cards                                         │
├──────────────────────────────────────────────────────┤
│ Input textarea                              [发送]   │
└──────────────────────────────────────────────────────┘
```

## 5.2 输入区

规则：

- Enter 发送；
- Shift + Enter 换行；
- 最大输入 500 字；
- 请求中显示取消按钮；
- 空输入禁止提交；
- 超时 20s 允许重试。

## 5.3 来源卡片

每条来源展示：

- 文章标题；
- 命中片段；
- 相关度标签；
- 跳转原文；
- 可选 chunk 序号。

普通用户显示“高相关 / 中相关 / 低相关”，调试模式显示 score。

# 六、后台登录 `/admin/login`

渲染模式：CSR  
目标：简单、可靠、安全提示明确。

## 6.1 原型

```txt
┌────────────────────────────┐
│ 智能内容平台 Admin          │
```

- 输入校验失败；
- 登录中；

- 服务不可用；
- 登录成功跳转 `/admin`。

# 七、文章管理 `/admin/articles`

渲染模式：CSR  
目标：真实后台列表体验，支持筛选、管理和向量同步。

## 7.1 原型

```txt
AdminLayout
  Sidebar
  Main
    PageHeader: 文章管理 [新建文章]
    FilterBar: 搜索 / 分类 / 标签 / 发布状态 / 向量状态
    DataTable
      标题 | 分类 | 标签 | 状态 | 向量状态 | 更新时间 | 操作
    Pagination
```

## 7.2 行操作

- 编辑；
- 预览；
- 同步向量；
- 删除。

删除必须二次确认；同步向量失败时保留失败状态和重试入口。

---

# 八、文章编辑 `/admin/articles/new`、`/admin/articles/[id]`

渲染模式：CSR  
目标：Markdown 编辑、预览、发布、向量同步一体化。

## 8.1 Desktop 原型

```txt
AdminLayout
  Header: 新建文章 / 编辑文章
  MetaForm: 标题 / 摘要 / slug / 分类 / 标签 / SEO
  Toolbar: 保存草稿 / 预览 / 发布
  EditorGrid
    Left: MarkdownEditor
    Right: MarkdownPreview
  BottomStatus: 保存状态 / 发布状态 / 向量同步状态
```

## 8.2 发布流程

```txt
编辑内容
→ 保存草稿
→ 发布前校验
→ 提交文章
→ 后端保存
→ 触发向量同步
→ 后台展示同步状态
```

## 8.3 校验

- 标题必填；
- slug 必填且唯一；
- 分类必选；
- 摘要不超过 160 字；
- 正文不为空；
- SEO title 不超过 60 字；
- SEO description 不超过 160 字。

## 9.1 原型

```txt

  Recent

```

# 十、监控大盘 `/admin/monitor`

渲染模式：CSR  
目标：展示前端监控 SDK 的采集价值。

## 10.1 原型

```txt
AdminLayout
  Filter: 时间范围 / 页面 / 事件类型
  WebVitalsChart
  PageViewChart
  ErrorTable
  ResourceErrorTable
  ExposureTable
```

## 10.2 空状态

如果还没有监控数据，展示：

```txt
暂无监控数据
访问前台页面后，SDK 会自动采集 PV、Web Vitals 和异常信息。
```

# 十一、知识库运维 `/admin/knowledge`

渲染模式：CSR  
目标：让 RAG 数据处理链路可视化。

## 11.1 原型

```txt
AdminLayout
  MetricGrid
    总文章数 / 总分片数 / 已同步 / 失败
  RebuildPanel
    [全量重建] [同步失败文章]
  ArticleVectorTable
    文章标题 | 分片数 | 向量状态 | 最近同步 | 操作
```

## 11.2 操作状态

- idle；
- rebuilding；
- success；
- failed；
- partial failed。

# 十二、关于页 `/about`

渲染模式：SSG / 预渲染  
目标：作为项目说明和面试展示页。

内容顺序：

1. 项目简介；
2. 技术栈；
3. 架构图；
4. 核心功能截图墙；
5. 性能与测试结果；
6. 项目亮点总结；
7. GitHub / Demo 链接。

截图墙首版可以先用占位图，开发完成后替换真实截图。
