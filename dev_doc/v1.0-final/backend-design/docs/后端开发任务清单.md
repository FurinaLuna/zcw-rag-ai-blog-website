# 二、Phase 0：项目骨架与基础设施

## 0.1 初始化 FastAPI 项目

任务：
- 创建 `backend/` FastAPI 项目（推荐使用 `uv` 或 `poetry` 管理依赖）
- 编写 `pyproject.toml` / `requirements.txt`
- 配置 `app/main.py` 应用入口（lifespan 管理）
- 配置 uvicorn 启动参数

建议依赖清单：
```
fastapi>=0.111.0
uvicorn[standard]>=0.30.0
sqlalchemy[asyncio]>=2.0.0
asyncpg>=0.29.0
pgvector>=0.3.0
alembic>=1.13.0
pydantic>=2.0.0
pydantic-settings>=2.0.0
python-jose[cryptography]>=3.3.0
passlib[bcrypt]>=1.7.0
redis[hiredis]>=5.0.0
httpx>=0.27.0
sentence-transformers>=2.7.0
loguru>=0.7.0
```

验收：
- `uvicorn app.main:app --reload` 可启动
- `/api/v1/health` 返回 200
- 自动生成 OpenAPI 文档（`/docs`）
- Python 依赖无冲突

## 0.2 环境配置与日志

任务：
- 实现 `app/core/config.py`（pydantic-settings）
- 创建 `.env.example`
- 实现 `app/core/logging.py`（loguru 配置）
- 配置开发/生产两套日志级别

验收：
- 环境变量可正常读取
- `.env` 文件可覆盖默认值
- 控制台和文件日志正常输出

## 0.3 数据库连接与 ORM

任务：
- 实现 `app/core/database.py`（async engine + session 工厂）
- 编写所有 ORM 模型（8 张表）
- 创建数据库基础模型类（`Base`，含 `id`、`created_at`、`updated_at` 混入）
- 配置 Alembic（`alembic init` + `env.py` 适配 async engine）

模型清单：
- `app/models/admin.py`
- `app/models/article.py`
- `app/models/category.py` (统一管理分类与专题)
- `app/models/tag.py`
- `app/models/article_tag.py`
- `app/models/comment.py`
- `app/models/monitor_log.py`
- `app/models/vector_chunk.py`

验收：
- `alembic upgrade head` 可创建全部表
- async session 可正常执行 CRUD
- 模型关系（ForeignKey）正确

## 0.4 鉴权基础

任务：
- 实现 `app/core/security.py`（bcrypt 哈希 + JWT 签发/校验）
- 实现 `app/dependencies/auth.py`（`get_current_admin` 依赖）
- 实现 `app/dependencies/db.py`（`get_db` session 依赖）

验收：
- `hash_password()` + `verify_password()` 正常
- JWT 签发后可正确解析
- 无效/过期 token 返回 401

## 0.5 中间件

任务：
- 实现 CORS 中间件（根据环境变量配置允许域名）
- 实现全局异常捕获中间件（统一错误响应格式）
- 实现请求日志中间件（loguru 记录 method/path/ip/duration）
- 实现接口限流中间件（基于 Redis 滑动窗口，可降级到内存）

验收：
- 跨域请求正常
- 未捕获的异常返回标准化错误
- 请求日志完整
- 超频请求返回 429

# 三、Phase 1：内容管理主链路

## 1.1 管理员登录

任务：
- 实现 `app/services/auth.py`（密码校验 + token 签发）
- 实现 `app/routers/admin.py` 中的登录路由
- 校验参数（用户名密码非空）

接口：
- `POST /api/v1/admin/login`
- `GET /api/v1/admin/me`

验收：
- 正确密码返回 token
- 错误密码返回 401
- 登录频率限制生效
- `/admin/me` 可获取当前管理员信息

## 1.2 分类与专题管理

任务：
- 实现 `app/services/category.py`
- 实现 `app/routers/admin.py` 中的分类/专题 CRUD 路由
- 实现 `app/routers/public.py` 中的分类与专题列表路由

接口：
- `GET /api/v1/public/categories`
- `GET /api/v1/public/topics`
- `GET /api/v1/public/topics/{slug}`
- `GET/POST/PUT/DELETE /api/v1/admin/categories`
- `PUT /api/v1/admin/categories/sort`

验收：
- 前台可获取分类/专题列表（含文章数）
- 后台可增删改查分类与专题（通过 `type` 字段区分）
- 关联文章的分类/专题不可删除
- 排序功能正常

## 1.3 标签管理

任务：
- 实现 `app/services/tag.py`
- 实现管理端和前台标签路由

接口：
- `GET /api/v1/public/tags`
- `GET /api/v1/public/tags/{tag_id}`
- `GET/POST/PUT/DELETE /api/v1/admin/tags`

验收：
- 标签 CRUD 正常
- 删除标签级联清理 article_tag

## 1.4 文章管理

任务：
- 实现 `app/services/article.py`（CRUD + 分页 + 筛选 + 搜索）
- 实现 `app/routers/admin.py` 中的文章管理路由
- 实现 `app/routers/public.py` 中的文章列表/详情路由
- 实现 slug 唯一性校验
- 发布前校验逻辑
- 浏览量统计（基于 Redis + IP 去重）

接口：
- `GET /api/v1/public/articles`（列表 + 筛选 + 分页）
- `GET /api/v1/public/articles/{slug}`（详情）
- `POST /api/v1/public/articles/{slug}/view`（浏览量）
- `GET/POST/PUT/DELETE /api/v1/admin/articles`
- `POST /api/v1/admin/articles/{id}/publish`
- `POST /api/v1/admin/articles/{id}/archive`
- `GET /api/v1/admin/articles/check-slug`

验收：
- 前台可按分类/专题/标签/关键词筛选文章
- 文章详情返回完整 Markdown 内容
- 前后台响应结构区分（后台含管理字段）
- slug 唯一性正确校验
- 浏览量 IP 去重生效

## 1.5 评论系统

任务：
- 实现 `app/services/comment.py`
- 实现提交评论和获取评论列表路由
- 实现评论点赞

接口：
- `GET /api/v1/public/articles/{slug}/comments`
- `POST /api/v1/public/articles/{slug}/comments`
- `POST /api/v1/public/comments/{id}/like`

验收：
- 匿名评论可提交（nickname + content）
- 评论可按时间排序
- 点赞数正确递增
- 空内容不可提交

## 1.6 搜索功能

任务：
- 实现 `app/services/article.py` 中的搜索逻辑（标题 + 摘要）
- PostgreSQL `ilike` 或 `to_tsvector` 全文检索

接口：
- `GET /api/v1/public/search?q=xxx`

验收：
- 可搜索到包含关键词的文章
- 空结果返回空列表
- 中文关键词可正常搜索

# 四、Phase 2：RAG 与监控闭环

## 2.1 RAG 数据处理链路

任务：
- 实现 `app/services/rag/cleaner.py`（Markdown 文本清洗）
- 实现 `app/services/rag/splitter.py`（规则分段切片）
- 实现 `app/services/rag/embedder.py`（BGE 模型加载 + 批量向量化）
- 实现 `app/services/rag/sync.py`（单篇同步 + 全量重建）

验收：
- 文本清洗去掉 Markdown 标记
- 分段大小在 50-200 字范围
- 向量化返回 512 维向量
- Embedder 模型正常加载（开发环境无 GPU 也可运行）
- 文章发布后自动触发向量同步

## 2.2 RAG 检索

任务：
- 实现 `app/services/rag/retriever.py`（pgvector TopK 检索）
- 实现相似度阈值过滤
- 实现简单权重重排

验收：
- 可检索到与问题相似的向量分片
- 低于阈值的结果被过滤
- 无匹配结果返回空列表

## 2.3 RAG 问答

任务：
- 实现 `app/services/rag/generator.py`（LLM API 调用）
- 实现系统提示词模板（边界约束）
- 实现流式输出（SSE，P1）
- 实现 `app/routers/rag.py` 问答路由
- 实现示例问题接口

接口：
- `POST /api/v1/rag/ask`
- `POST /api/v1/rag/ask/stream`（P1）
- `GET /api/v1/rag/suggestions`

验收：
- 可提交问题并获取回答
- 回答附引用来源（文章标题 + 命中片段 + 相关度）
- 无匹配时返回兜底回复
- LLM API 异常时返回友好提示
- 接口限流生效

## 2.4 监控数据上报

任务：
- 实现 `app/services/monitor.py`（事件批量写入）
- 实现 `app/routers/monitor.py` 上报路由
- 事件校验（type 必须是合法枚举值）
- 实现采样逻辑（生产环境对 normal 事件按比例丢弃）

接口：
- `POST /api/v1/monitor/report`

验收：
- 单条/批量事件可正常写入
- 非法事件类型被拒绝
- 监控日志表写入正常
- 上报接口吞吐量达标（≥300/分钟无性能问题）

## 2.5 监控数据查询

任务：
- 实现监控数据分页查询
- 实现按事件类型、页面路径、时间范围筛选

接口：
- `GET /api/v1/monitor/stats`

验收：
- 可按条件查询监控日志
- 分页正常

# 七、建议开发顺序

```
1. FastAPI 骨架 + config + 日志 + health
2. ORM 模型 + Alembic 迁移 + 种子数据
3. 鉴权（JWT + bcrypt + 中间件）
4. 分类/专题/标签 CRUD（最简单的先做）
5. 前台接口（文章列表/详情/搜索）
6. 后台文章管理 CRUD
7. 评论系统
8. RAG 数据处理链路（清洗/切片/向量化/检索）
9. RAG 问答接口
10. 监控数据上报
11. 运营大盘 + 知识库运维
12. 测试 + Docker 部署 + 文档
```

原则：先打通"前台能看、后台能写"的主链路，再接入 RAG 和监控等复杂模块，最后做测试和部署。
