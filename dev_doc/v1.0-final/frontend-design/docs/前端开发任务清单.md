# 二、Phase 0：项目骨架与基础工程

## 0.1 初始化 Nuxt3 项目

任务：

- 创建 `frontend/` Nuxt3 项目；
- 配置 TypeScript；
- 配置 TailwindCSS；
- 配置 Pinia；
- 配置 ESLint / Prettier；
- 配置基础目录结构。

建议目录：

```txt
frontend/
  assets/
  components/
  composables/
  directives/
  layouts/
  middleware/
  pages/
  plugins/
  stores/
  styles/
  types/
```

验收：

- `npm run dev` 可启动；
- 首页能正常访问；
- TypeScript 无基础报错；
- Tailwind 样式生效。

## 0.2 全局样式与设计 Token

任务：

- 创建 `styles/tokens.css`；
- 创建 `styles/base.css`；
- 创建 `styles/global.css`；
- 写入颜色、间距、圆角、focus-visible、reduced-motion。

验收：

- 全局背景、文本色、链接色生效；
- 键盘 Tab 可见明显焦点；
- `prefers-reduced-motion` 生效。

## 0.3 基础布局

任务：

- 实现 `layouts/default.vue`；
- 实现 `layouts/admin.vue`；
- 实现 `SiteHeader`、`SiteFooter`、`SiteNav`、`SkipToContent`；
- 实现 `AdminSidebar`、`AdminTopbar`。

验收：

- 前台页面使用 `default` 布局；
- 后台页面使用 `admin` 布局；
- 移动端导航可折叠；
- 后台侧边栏在桌面端可用。

## 0.4 请求层与环境配置

任务：

- 编写前端 `.env.example` 和 `.env` 配置；
- 实现 `useApi.ts`；
- 封装 `$fetch` baseURL (区分客户端和服务端请求地址)；
- 支持 JWT 注入；
- 支持统一错误转换；
- 根据后端 FastAPI 生成 OpenAPI 类型定义（`api.d.ts`）。

验收：

- 前端可请求后端健康检查接口；
- 401 能跳转登录；
- 错误能统一展示；
- 接口传参和返回值有 TS 类型推导。

# 七、建议开发顺序

```txt
1. Nuxt3 骨架 + 样式 token + 布局 + OpenAPI 类型与请求封装
2. Mock 数据驱动首页 / 文章列表 / 文章详情 / 专题
3. 接入真实后端文章 API
4. 后台登录 + 文章管理 + Markdown 编辑器
5. RAG 问答页 + 引用来源展示
6. 前端监控 SDK 基础采集
7. Dashboard / 监控 / 知识库运维
8. 测试、部署、截图、README
```

原则：先用 mock 数据完成前端体验，再逐步接入后端接口，避免前端开发被后端进度阻塞。
