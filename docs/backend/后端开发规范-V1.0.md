# 一、后端架构总览

```
┌─ API Layer ────────────────────────────────────────┐
│  routers/: 前台 / 后台 / AI / 监控 四组路由          │
│  dependencies/: 鉴权依赖、数据库会话依赖、参数校验      │
│  middleware/: CORS、限流、异常统一处理、请求日志        │
├─ Service Layer ────────────────────────────────────┤
│  services/: 文章/分类/标签/评论/监控 CRUD             │
│  services/rag/: 文本清洗、切片、向量化、检索、问答     │
│  schemas/: Pydantic 请求/响应模型                    │
├─ Data Layer ───────────────────────────────────────┤
│  models/: SQLAlchemy ORM 模型                       │
│  migrations/: Alembic 迁移脚本                       │
│  repositories/: 数据访问封装（可选，轻量项目可跳过）    │
├─ Infrastructure ───────────────────────────────────┤
│  core/: 配置管理、安全工具、日志                       │
│  utils/: 辅助工具（文本清洗、时间处理等）              │
│  tests/: pytest 单元测试 + 集成测试                   │
└─────────────────────────────────────────────────────┘
```

分层原则：
- API Layer 只负责路由注册、参数提取、响应返回，不包含业务逻辑
- Service Layer 承载所有业务逻辑，Service 之间可互相调用
- Data Layer 通过 SQLAlchemy 2.0 async session 访问数据库
- Infrastructure 层提供横切能力（配置、日志、安全），通过依赖注入使用

# 六、鉴权设计

## 6.1 JWT 流程

1. 管理员通过 `/api/v1/admin/login` 提交账号密码
2. 服务端校验密码（bcrypt），签发 access token（JWT，有效期 2 小时）
3. 前端在请求头携带 `Authorization: Bearer <token>`
4. 后台接口通过 `get_current_admin` 依赖校验 token
5. Token 过期返回 401，前端跳转登录页

## 6.2 JWT 实现

```python
# app/core/security.py
from datetime import datetime, timedelta, timezone
from jose import jwt, JWTError
from passlib.context import CryptContext

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def verify_password(plain: str, hashed: str) -> bool:
    return pwd_context.verify(plain, hashed)

def hash_password(password: str) -> str:
    return pwd_context.hash(password)

def create_access_token(data: dict, expires_delta: timedelta = timedelta(hours=2)) -> str:
    to_encode = data.copy()
    to_encode.update({"exp": datetime.now(timezone.utc) + expires_delta})
    return jwt.encode(to_encode, settings.secret_key, algorithm="HS256")
```

## 6.3 鉴权依赖

```python
# app/dependencies/auth.py
from fastapi import Depends, HTTPException, status
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials
from jose import jwt, JWTError
from app.core.config import settings

security = HTTPBearer()

async def get_current_admin(
    credentials: HTTPAuthorizationCredentials = Depends(security),
) -> dict:
    try:
        payload = jwt.decode(credentials.credentials, settings.secret_key, algorithms=["HS256"])
        return payload
    except JWTError:
        raise HTTPException(status_code=status.HTTP_401_UNAUTHORIZED, detail="无效的认证令牌")
```

# 七、RAG 数据处理链路

## 7.1 整体流程

```
文章发布/更新
  → 文本清洗 (cleaner.py)
    → 规则分段切片 (splitter.py)
      → BGE 向量化 (embedder.py)
        → pgvector 存储 (vector_chunk 表)

用户提问

    → 问题向量化
      → pgvector TopK 检索 (retriever.py)
        → 权重重排
          → 上下文拼接 + 系统提示词
            → LLM 生成回答 (generator.py)
              → 流式返回前端
```

## 7.2 文本清洗（cleaner.py）

```python
import re

def clean_text(text: str) -> str:
    """清洗文章 Markdown 正文，输出纯文本用于切片"""
    # 移除 Markdown 语法但保留文本
    text = re.sub(r"```[\s\S]*?```", "", text)  # 移除代码块
    text = re.sub(r"\[([^\]]*)\]\([^\)]*\)", r"\1", text)  # 链接保留文字
    text = re.sub(r"[#*`>~\-|]", "", text)  # 移除 Markdown 标记
    text = re.sub(r"\n{3,}", "\n\n", text)  # 合并多个换行
    text = re.sub(r"\s{2,}", " ", text)  # 合并多余空格
    return text.strip()
```

## 7.3 分段切片（splitter.py）

```python
import re
from typing import List

def split_text(text: str, chunk_size: int = 120, overlap: int = 20) -> List[str]:
    """
    规则分段：优先按段落切分，段落过长按句子切分，句子过长按 chunk_size 截断。
    chunk_size 120 字（约 240 token），overlap 20 字保证上下文连续。
    """
    paragraphs = [p.strip() for p in text.split("\n\n") if p.strip()]
    chunks = []

    for para in paragraphs:
        if len(para) <= chunk_size:
            chunks.append(para)
        else:
            sentences = re.split(r"[。！？；\n]", para)
            current = ""
            for sent in sentences:
                sent = sent.strip()
                if not sent:
                    continue
                if len(current) + len(sent) <= chunk_size:
                    current += sent + "。"
                else:
                    if current:
                        chunks.append(current.strip())
                    # 重叠处理
                    overlap_text = current[-overlap:] if current else ""
                    current = overlap_text + sent + "。"

            if current.strip():
                chunks.append(current.strip())

    return chunks
```

## 7.4 向量化（embedder.py）

```python
from sentence_transformers import SentenceTransformer
from typing import List

# 全局单例，避免重复加载模型
_model: SentenceTransformer | None = None

def get_embedder(model_name: str = "BAAI/bge-small-zh-v1.5") -> SentenceTransformer:
    global _model
    if _model is None:
        _model = SentenceTransformer(model_name)
    return _model

def embed_texts(texts: List[str]) -> List[List[float]]:
    """批量文本向量化，返回 512 维向量列表"""
    model = get_embedder()
    embeddings = model.encode(texts, normalize_embeddings=True)
    return embeddings.tolist()

def embed_query(query: str) -> List[float]:
    """查询向量化，BGE 模型查询需要加前缀"""
    model = get_embedder()
    embedding = model.encode(f"为这个句子生成表示以用于检索相关文章：{query}", normalize_embeddings=True)
    return embedding.tolist()
```

说明：embedding 模型首版直接集成在 FastAPI 进程中加载。如果模型加载时间过长或显存不足，可拆分为独立 embedding 微服务，通过 HTTP/gRPC 调用。但首版为降低部署复杂度，进程内集成即可。

## 7.5 检索（retriever.py）

```python
from sqlalchemy import text
from sqlalchemy.ext.asyncio import AsyncSession
from typing import List

async def retrieve_similar_chunks(
    db: AsyncSession,
    query_embedding: List[float],
    top_k: int = 5,
    similarity_threshold: float = 0.7,
) -> List[dict]:
    """
    pgvector 余弦相似度检索 TopK 相关分片。
    返回 list[{"chunk_id": int, "article_id": int, "content": str, "similarity": float}]
    """
    query = text("""
        SELECT
            vc.id AS chunk_id,
            vc.article_id,
            vc.content,
            1 - (vc.embedding <=> :query_emb) AS similarity
        FROM vector_chunk vc
        WHERE 1 - (vc.embedding <=> :query_emb) >= :threshold
        ORDER BY vc.embedding <=> :query_emb
        LIMIT :top_k
    """)
    result = await db.execute(query, {
        "query_emb": str(query_embedding),  # pgvector 接受字符串形式的向量
        "threshold": similarity_threshold,
        "top_k": top_k,
    })
    rows = result.fetchall()
    return [
        {"chunk_id": r.chunk_id, "article_id": r.article_id,
         "content": r.content, "similarity": round(r.similarity, 4)}
        for r in rows
    ]

def rerank_by_article_popularity(chunks: List[dict], article_stats: dict) -> List[dict]:
    """
    简单权重重排：检索相似度 0.7 + 文章热度归一化 0.3。
    article_stats: {article_id: popularity_score}
    """
    if not article_stats:
        return chunks
    max_pop = max(article_stats.values()) or 1
    for chunk in chunks:
        pop_score = article_stats.get(chunk["article_id"], 0) / max_pop
        chunk["score"] = round(chunk["similarity"] * 0.7 + pop_score * 0.3, 4)
    return sorted(chunks, key=lambda x: x["score"], reverse=True)
```

# 八、接口设计原则

## 8.1 RESTful 规范

- URL 前缀：`/api/v1`
- 前台接口：`/api/v1/public/*`（无需鉴权）
- 后台接口：`/api/v1/admin/*`（需 JWT 鉴权）
- 监控上报：`/api/v1/monitor/*`（无需鉴权，仅做来源校验）
- AI 问答：`/api/v1/rag/*`（无需鉴权，但需限流）

## 8.2 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

分页响应：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [...],
    "total": 100,
    "page": 1,
    "page_size": 20
  }
}
```

错误响应：
```json
{
  "code": 40401,
  "message": "文章不存在",
  "detail": null
}
```

错误码规范：
- `400xx`：客户端参数错误
- `40100`：未认证
- `40300`：无权限
- `404xx`：资源不存在
- `409xx`：资源冲突（如 slug 重复）
- `42200`：参数校验失败
- `42900`：请求限流
- `50000`：服务器内部错误
- `50200`：上游服务（LLM API）不可用

# 九、接口限流设计

| 接口 | 限流策略 | 说明 |
|------|---------|------|
| `/api/v1/public/*` | 100次/分钟/IP | 内容读取，宽松限流 |
| `/api/v1/rag/ask` | 10次/分钟/IP | AI 问答，防止资源滥用 |
| `/api/v1/admin/login` | 5次/分钟/IP | 登录接口，防止暴力破解 |
| `/api/v1/admin/*` | 60次/分钟 | 后台操作，管理员使用 |
| `/api/v1/monitor/report` | 300次/分钟 | 监控上报，高吞吐 |

实现方式：基于 Redis 的滑动窗口计数器。Redis 不可用时降级为内存计数器（单进程有效）。

使用 loguru 结构化日志：

```python
from loguru import logger

logger.error("rag_generation_failed", query="...", error=str(e))

# RAG 检索日志

```

日志级别：
- DEBUG：开发环境详细日志
- INFO：生产环境请求/业务关键节点
- WARNING：降级、重试、非关键异常
- ERROR：需要关注的错误（接口异常、LLM 调用失败等）
