# 三、分类与专题表 `category`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PK | 主键 |
| name | VARCHAR(50) | NOT NULL | 分类/专题名称 |
| slug | VARCHAR(100) | NOT NULL, UNIQUE | URL 友好的唯一标识 |
| type | VARCHAR(20) | NOT NULL, DEFAULT 'category' | category / topic |
| description | VARCHAR(200) | — | 描述 |
| cover_url | VARCHAR(500) | — | 封面图 URL（主要用于专题） |
| sort_order | INTEGER | NOT NULL, DEFAULT 0 | 排序权重（越小越前） |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | 创建时间 |

索引：
- `idx_category_slug` UNIQUE ON (slug)
- `idx_category_type` ON (type)
- `idx_category_sort` ON (sort_order)

创建语句：
```sql
CREATE TABLE category (
    id SERIAL PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    slug VARCHAR(100) NOT NULL UNIQUE,
    type VARCHAR(20) NOT NULL DEFAULT 'category'
        CHECK (type IN ('category', 'topic')),
    description VARCHAR(200),
    cover_url VARCHAR(500),
    sort_order INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_category_type ON category(type);
```

# 五、文章表 `article`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PK | 主键 |
| title | VARCHAR(200) | NOT NULL | 文章标题 |
| summary | VARCHAR(160) | — | 文章摘要（SEO description 可复用） |
| slug | VARCHAR(200) | NOT NULL, UNIQUE | URL 友好的唯一标识 |
| content_md | TEXT | — | Markdown 正文 |
| cover_url | VARCHAR(500) | — | 封面图 URL |
| category_id | INTEGER | FK → category(id), ON DELETE SET NULL | 分类/专题 ID |
| status | VARCHAR(20) | NOT NULL, DEFAULT 'draft' | draft / published / archived |
| vector_status | VARCHAR(20) | NOT NULL, DEFAULT 'pending' | pending / syncing / synced / failed |
| view_count | INTEGER | NOT NULL, DEFAULT 0 | 浏览量计数 |
| seo_title | VARCHAR(60) | — | 自定义 SEO 标题 |
| seo_description | VARCHAR(160) | — | 自定义 SEO 描述 |
| published_at | TIMESTAMPTZ | — | 首次发布时间 |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | 更新时间 |

> `reading_time` 为 API 层计算字段，不存储在数据库中。计算公式：`max(1, ceil(len(content_md) / 500))`。

索引：
- `idx_article_slug` UNIQUE ON (slug)
- `idx_article_status` ON (status)
- `idx_article_category` ON (category_id)
- `idx_article_published` ON (published_at DESC) WHERE status = 'published'
- `idx_article_vector_status` ON (vector_status)

创建语句：
```sql
CREATE TABLE article (
    id SERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    summary VARCHAR(160),
    slug VARCHAR(200) NOT NULL UNIQUE,
    content_md TEXT,
    cover_url VARCHAR(500),
    category_id INTEGER REFERENCES category(id) ON DELETE SET NULL,
    status VARCHAR(20) NOT NULL DEFAULT 'draft'
        CHECK (status IN ('draft', 'published', 'archived')),
    vector_status VARCHAR(20) NOT NULL DEFAULT 'pending'
        CHECK (vector_status IN ('pending', 'syncing', 'synced', 'failed')),
    view_count INTEGER NOT NULL DEFAULT 0,
    seo_title VARCHAR(60),
    seo_description VARCHAR(160),
    published_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_article_published ON article (published_at DESC) WHERE status = 'published';
CREATE INDEX idx_article_category ON article (category_id);
CREATE INDEX idx_article_vector_status ON article (vector_status);
```

# 八、埋点日志表 `monitor_log`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGSERIAL | PK | 主键（大数据量用 BIGINT） |
| event_type | VARCHAR(30) | NOT NULL | pv / duration / web_vital / error / resource_error / api_error / exposure |
| page_url | VARCHAR(500) | NOT NULL | 页面路径 |
| event_data | JSONB | NOT NULL | 事件数据（灵活 schema） |
| session_id | VARCHAR(100) | — | 会话标识 |
| client_ip | VARCHAR(45) | — | 客户端 IP |
| user_agent | VARCHAR(500) | — | User-Agent |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | 采集时间 |

索引：
- `idx_monitor_type_time` ON (event_type, created_at DESC)
- `idx_monitor_page_time` ON (page_url, created_at DESC)
- `idx_monitor_created` ON (created_at DESC)

设计说明：监控日志表采用 JSONB 存储事件数据，避免为每种事件类型创建独立字段，同时保留灵活的查询能力。大批量写入场景按月分区（首版可省略，二期做）。

创建语句：
```sql
CREATE TABLE monitor_log (
    id BIGSERIAL PRIMARY KEY,
    event_type VARCHAR(30) NOT NULL,
    page_url VARCHAR(500) NOT NULL,
    event_data JSONB NOT NULL,
    session_id VARCHAR(100),
    client_ip VARCHAR(45),
    user_agent VARCHAR(500),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_monitor_type_time ON monitor_log (event_type, created_at DESC);
CREATE INDEX idx_monitor_page_time ON monitor_log (page_url, created_at DESC);
CREATE INDEX idx_monitor_created ON monitor_log (created_at DESC);
```

event_data 按事件类型的 JSONB 结构：

```json
// pv
{"referrer": "https://www.google.com"}

// duration
{"duration_ms": 45000}

// web_vital
{"metric": "LCP", "value": 1234.5, "rating": "good"}

// error (JS/Promise)
{"error_type": "js_error", "message": "...", "filename": "...", "lineno": 42, "colno": 15, "stack": "..."}

// resource_error
{"resource_type": "img", "resource_url": "...", "element_tag": "img"}

// api_error
{"method": "GET", "url": "/api/v1/public/articles", "status": 500, "duration_ms": 1200}

// exposure
{"element_id": "article-card-1", "element_type": "article_card", "duration_ms": 3500}
```

# 九、向量分片表 `vector_chunk`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | SERIAL | PK | 主键 |
| article_id | INTEGER | FK → article(id), ON DELETE CASCADE | 关联文章 |
| content | TEXT | NOT NULL | 分片文本内容 |
| embedding | vector(512) | NOT NULL | BGE-small-zh-v1.5 嵌入向量（512维） |
| chunk_index | INTEGER | NOT NULL | 分片在文章内的序号 |
| created_at | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | 创建时间 |

索引：
- `idx_vector_article` ON (article_id)
- `idx_vector_embedding` IVFFLAT 向量索引

创建语句：
```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE vector_chunk (
    id SERIAL PRIMARY KEY,
    article_id INTEGER NOT NULL REFERENCES article(id) ON DELETE CASCADE,
    content TEXT NOT NULL,
    embedding vector(512) NOT NULL,
    chunk_index INTEGER NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_vector_article ON vector_chunk (article_id);
CREATE INDEX idx_vector_embedding ON vector_chunk
    USING ivfflat (embedding vector_cosine_ops)
    WITH (lists = 100);
```

IVFFLAT 索引说明：
- `lists = 100`：适用于 1000-10000 条向量的场景，首版数据量使用此值
- 如果向量数量到 10 万级别，建议 `lists = sqrt(rows)`
- 索引在批量写入数据后创建，写入前先 Drop 索引，写入完成后重建

# 十、向量相似度检索 SQL

## 10.1 余弦相似度检索（推荐）

```sql
SELECT
    vc.id AS chunk_id,
    vc.article_id,
    vc.content,
    vc.chunk_index,
    1 - (vc.embedding <=> :query_embedding) AS similarity
FROM vector_chunk vc
WHERE 1 - (vc.embedding <=> :query_embedding) >= :threshold
ORDER BY vc.embedding <=> :query_embedding
LIMIT :top_k;
```

## 10.2 结合文章热度重排

```sql
SELECT
    vc.id AS chunk_id,
    vc.article_id,
    vc.content,
    1 - (vc.embedding <=> :query_embedding) AS similarity,
    a.view_count
FROM vector_chunk vc
JOIN article a ON vc.article_id = a.id
WHERE a.status = 'published'
    AND 1 - (vc.embedding <=> :query_embedding) >= :threshold
ORDER BY vc.embedding <=> :query_embedding
LIMIT :top_k * 2;  -- 多取2倍用于重排
```

```python
        sa.Column("created_at", sa.DateTime(timezone=True), server_default=sa.text("now()"), nullable=False),
        sa.Column("last_login_at", sa.DateTime(timezone=True), nullable=True),
        sa.PrimaryKeyConstraint("id"),

    )

def downgrade() -> None:
```

## 11.3 迁移命令

