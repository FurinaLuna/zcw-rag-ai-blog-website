# 智能内容平台前端组件规范

版本：V1.0  
日期：2026-05-03  
关联文档：`前端设计规范-V1.0.md`  
目标：补齐按钮、表单、卡片、表格、状态、反馈、图表等组件级规范，保证 Nuxt3 前端实现时视觉一致、交互可控、可访问性达标。

---

# 一、组件设计原则

1. 内容优先：组件服务阅读、检索、编辑和监控，不做无意义装饰。
2. 密度适中：前台保持舒适阅读密度，后台保持高信息密度但不拥挤。
3. 状态完整：每个数据组件必须覆盖 loading、empty、error、success、disabled。
4. 可访问：所有可点击元素使用真实 button/link，支持键盘聚焦和 aria 标签。
5. 稳定尺寸：按钮、状态标签、表格行、图表容器必须有稳定高度，避免加载后布局跳动。
6. 圆角克制：卡片、输入框、按钮默认圆角不超过 8px。

---

# 二、设计 Token

## 2.1 间距

| Token | 值 | 用途 |
|------|----|------|
| `--space-1` | 4px | 图标与文字间距、紧凑元素 |
| `--space-2` | 8px | 表单内部、标签间距 |
| `--space-3` | 12px | 卡片内小间距 |
| `--space-4` | 16px | 常规组件 padding |
| `--space-6` | 24px | 区块间距 |
| `--space-8` | 32px | 页面模块间距 |
| `--space-12` | 48px | 首页大模块间距 |

## 2.2 圆角

| Token | 值 | 用途 |
|------|----|------|
| `--radius-sm` | 4px | Badge、代码复制按钮 |
| `--radius-md` | 6px | 输入框、普通按钮 |
| `--radius-lg` | 8px | 卡片、弹窗、问答气泡 |
| `--radius-full` | 999px | 头像、圆形图标按钮 |

## 2.3 阴影

首版尽量不用重阴影，通过边框和背景层级区分模块。

| Token | 值 | 用途 |
|------|----|------|
| `--shadow-focus` | `0 0 0 3px rgba(15, 118, 110, .18)` | 聚焦态 |
| `--shadow-popover` | `0 12px 30px rgba(31, 41, 51, .12)` | 下拉、浮窗 |

# 三、按钮

## 3.1 类型

| 类型 | 用途 | 样式 |
|------|------|------|
| Primary | 主操作，如发布、提交问题、登录 | 青绿色背景，白字 |
| Secondary | 次级操作，如预览、保存草稿 | 白底，边框，深色字 |
| Ghost | 导航、轻量操作 | 透明背景，hover 显示浅底 |
| Danger | 删除、危险操作 | 红色文字或红色边框 |
| Icon | 复制、搜索、关闭、刷新 | 仅图标，必须有 aria-label 或 tooltip |

## 3.2 尺寸

| 尺寸 | 高度 | Padding | 场景 |
|------|------|---------|------|
| sm | 32px | 8px 12px | 表格行操作 |
| md | 40px | 12px 16px | 常规按钮 |
| lg | 48px | 16px 20px | 首页问答入口、登录按钮 |
| icon | 36px * 36px | 0 | 图标按钮 |

## 3.3 状态

必须实现：

- default
- hover
- active
- focus-visible
- disabled
- loading

loading 状态下保留按钮原宽度，避免文字切换造成布局抖动。

# 四、表单

## 4.1 输入框

统一高度：

- 普通输入框：40px
- 搜索框：44px
- 文本域：最小 120px
- Markdown 编辑器：最小 520px

必须包含：

- label
- placeholder
- helper text
- error text
- disabled state
- focus-visible state

错误文案靠近字段展示，不只用红色边框表达错误。

## 4.2 Select / Combobox

用于分类、标签、状态筛选。

要求：

- 支持键盘上下选择；
- 支持搜索时展示空状态；
- 多选标签需要可删除；
- 下拉层宽度至少等于触发器宽度；
- 下拉层最大高度 320px，超出滚动。

## 4.3 Markdown 编辑器

布局：

```txt
Desktop: 左编辑 / 右预览
Tablet: 上编辑 / 下预览
Mobile: 默认只显示编辑，使用 Tab 切换预览
```

工具栏：

- 保存草稿
- 发布
- 预览切换
- 插入代码块
- 插入图片
- 同步向量状态

状态：

- 未保存
- 保存中
- 已保存
- 发布中
- 发布成功
- 发布失败
- 向量同步中
- 向量同步失败

# 五、卡片

## 5.1 ArticleCard

内容：

- 标题
- 摘要
- 标签
- 发布时间
- 阅读时间
- 可选小封面

约束：

- 标题最多 2 行，超出省略；
- 摘要最多 3 行；
- 卡片 hover 只改变边框和标题颜色，不做大幅移动；
- 封面图比例固定 16:9 或 4:3，避免布局跳动。

## 5.2 MetricCard

用于首页系统能力指标和后台 Dashboard。

内容：

- 指标名称
- 当前值
- 趋势/状态
- 可选说明

状态色：

- 正常：青绿色
- 警告：琥珀色
- 异常：红色
- 无数据：灰色

## 5.3 TopicCard

内容：

- 专题名称
- 简短描述
- 文章数
- 最近更新时间

视觉上比 ArticleCard 更轻，不使用大封面。

# 六、表格

后台列表使用通用 `DataTable`。

## 6.1 基础结构

必须支持：

- 表头固定；
- 行 hover；
- 空状态；
- 加载骨架；
- 错误重试；
- 分页；
- 批量选择（P1 可做）；
- 行操作菜单。

## 6.2 行高

- 常规行：52px
- 紧凑行：44px
- 表头：44px

## 6.3 文章表格列

```txt
标题 | 分类 | 标签 | 状态 | 向量状态 | 更新时间 | 操作
```

操作：

- 编辑
- 预览
- 同步向量
- 删除

删除必须二次确认。

# 七、状态标签 Badge

## 7.1 文章状态

| 状态 | 文案 | 颜色 |
|------|------|------|
| draft | 草稿 | 灰色 |
| published | 已发布 | 青绿色 |
| archived | 已归档 | 深灰 |

## 7.2 向量状态

| 状态 | 文案 | 颜色 |
|------|------|------|
| pending | 待同步 | 琥珀色 |
| syncing | 同步中 | 蓝色 |
| synced | 已同步 | 青绿色 |
| failed | 同步失败 | 红色 |

## 7.3 监控状态

| 状态 | 文案 | 颜色 |
|------|------|------|
| good | 良好 | 青绿色 |
| needs-improvement | 待优化 | 琥珀色 |
| poor | 异常 | 红色 |
| no-data | 无数据 | 灰色 |

# 八、反馈状态

## 8.1 Loading

使用场景：

- 首屏数据：骨架屏；
- 按钮提交：按钮内 spinner；
- 图表加载：固定高度 skeleton；
- RAG 回答：逐步输出或打字状态。

## 8.2 Empty

空状态必须给出下一步动作。

示例：

```txt
还没有文章
[新建文章]
```

```txt
没有找到相关知识片段
换个问题试试，或浏览最新文章。
```

## 8.3 Error

错误状态必须包含：

- 发生了什么；
- 是否可重试；
- 追踪 ID（可选，用于监控日志关联）；
- 不暴露原始堆栈。

# 九、RAG 问答组件

## 9.1 RAGChat

结构：

```txt
Header: 标题 + 关闭按钮
Body: 消息列表
SourcePanel: 引用来源
Input: 输入框 + 发送按钮
```

要求：

- 打开时聚焦输入框；
- Esc 关闭浮窗；
- Enter 发送，Shift+Enter 换行；
- 请求中可取消；
- 回答完成后展示引用来源；
- 来源点击跳转文章详情并定位到片段。

## 9.2 RAGSourceCard

内容：

- 文章标题
- 匹配片段
- 相关度
- 跳转链接

相关度不直接暴露小数给普通用户，可显示为：

- 高相关
- 中相关
- 低相关

调试模式下再展示具体 score。

# 十、图表

图表使用 ECharts，但外观保持克制。

## 10.1 图表容器

必须固定高度：

- 小图：220px
- 中图：320px
- 大图：420px

SSR 阶段渲染固定高度占位，客户端 mounted 后初始化图表。

## 10.2 图表类型

| 场景 | 图表 |
|------|------|
| PV/UV趋势 | 折线图 |
| Web Vitals趋势 | 折线图 |
| 文章排行 | 条形图 |
| 错误类型分布 | 横向条形图 |
| 向量同步状态 | 环形图或状态卡片 |

图表不使用大面积渐变，不使用 3D 图。

# 十一、导航与布局组件

## 11.1 SiteHeader

内容：

- Logo / 小头像
- 主导航：文章、专题、搜索、问答、关于
- 移动端菜单按钮

要求：

- sticky 可选，但不能遮挡阅读内容；
- 移动端菜单打开时锁定背景滚动；
- 当前路由高亮。

## 11.2 AdminSidebar

内容：

- Dashboard
- 文章管理
- 分类管理
- 标签管理
- 知识库运维
- 监控数据

要求：

- Desktop 固定左侧；
- Tablet 可折叠；
- Mobile 后台可不做完整适配，首版至少保证可横向滚动或提示桌面访问。

---

# 十二、可访问性要求

组件实现必须满足：

- 所有 IconButton 必须有 `aria-label`；
- 弹窗打开后焦点进入弹窗，关闭后回到触发按钮；
- 错误信息使用 `aria-live="polite"`；
- loading 使用 `aria-busy`；
- 表格有正确的 `th` 和 scope；
- 颜色不是唯一状态表达方式；
- focus-visible 样式必须明显。
