# 智能内容平台前端设计草稿

版本：V0.1 草稿版  
目标：为「基于Nuxt3混合渲染与PgVector向量检索的智能内容平台」提供前端页面、信息架构、视觉风格与组件拆分依据。  
定位：偏前端全栈暑期实习简历项目，重点展示内容体验、Nuxt3混合渲染、RAG交互、前端监控与后台运营能力。

---

# 一、产品定位

本项目不做传统个人博客，也不做重型企业后台，而是做一个「技术内容站 + 智能知识库 + 前端工程化展示台」。

用户打开网站后，应当直观看到三件事：

1. 这是一个真实可读的技术内容站；
2. 站内内容可以通过RAG智能问答检索；
3. 项目具备前端工程化深度，包括混合渲染、性能监控、异常采集、后台管理与可视化。

核心关键词：

- 技术写作
- 智能检索
- Nuxt3混合渲染
- Markdown阅读体验
- 前端监控SDK
- 内容后台管理
- RAG引用来源

# 二、设计基调

## 2.1 视觉方向

整体采用「安静、克制、技术编辑部」风格，介于技术博客、知识库和轻量工程控制台之间。

避免：

- 大面积紫色渐变；
- 玻璃拟态堆叠；
- 无意义科技背景图；
- 过度营销式首页；
- 只有卡片堆叠、没有内容密度的布局。

推荐：

- 浅色纸张背景或深色代码编辑器风格；
- 清晰的正文排版；
- 稳定的网格系统；
- 小面积强调色；
- 真实文章、真实数据、真实截图；
- 少量精准动效。

## 2.2 色彩建议

首版建议使用浅色主题，后续再加暗色模式。

- 背景色：`#F7F5F0` / `#FAFAF7`
- 主文本：`#1F2933`
- 次级文本：`#667085`
- 边框：`#E4E0D8`
- 强调色：青绿色 `#0F766E`
- 辅助强调色：琥珀色 `#B7791F`
- 代码背景：`#111827`

色彩原则：整体保持低饱和，强调色只用于链接、按钮、状态、图表关键数据。

## 2.3 字体建议

中文正文优先清晰可读，标题可以稍微有性格。

- 中文正文：系统默认中文字体或思源黑体类字体；
- 英文/代码：`JetBrains Mono` 或 `Fira Code`；
- 标题：可以使用更有编辑部气质的衬线字体，但不要影响可读性。

# 三、信息架构

## 3.1 前台路由

```txt
/                    首页
/articles             文章列表
/articles/[slug]      文章详情
/topics               专题聚合
/topics/[slug]        专题详情
/search               搜索页
/ask                  站内智能问答
/about                关于与项目说明
```

## 3.2 后台路由

```txt
/admin/login          后台登录
/admin                运营大盘
/admin/articles       文章管理
/admin/articles/new   新建文章
/admin/articles/[id]  编辑文章
/admin/categories     分类管理
/admin/tags           标签管理
/admin/knowledge      知识库运维
/admin/monitor        前端监控数据
```

## 3.3 渲染模式建议

```txt
/                    SSG/预渲染
/articles             SSG/预渲染
/articles/[slug]      SSR
/topics               SSG/预渲染
/topics/[slug]        SSG或SSR
/search               SSR或CSR
/ask                  CSR
/about                SSG/预渲染
/admin/**             CSR
```

依赖浏览器API的组件必须使用 `.client.vue` 或 `onMounted` 隔离，例如：

- `RAGChat.client.vue`
- `TrackerBootstrap.client.vue`
- `ExposureObserver.client.vue`
- `ThemeToggle.client.vue`
- `MarkdownToc.client.vue`

# 四、首页设计

首页第一屏不做空泛大Hero，而是直接展示真实内容和系统能力。

## 4.1 首页首屏

首屏结构：

- 左侧：站点名称、定位、当前关注方向；
- 右侧：迷你RAG问答框；
- 底部：最近文章和系统指标露出。

建议文案：

```txt
智能技术知识库
记录前端工程化、Nuxt实践、RAG应用与性能监控。
```

标签示例：

- Frontend Engineering
- Nuxt / Vue
- RAG
- Performance
- Monitoring

## 4.2 首页模块

1. 精选文章

展示3到5篇文章，包含标题、摘要、标签、阅读时间、发布日期、可选小封面。

示例文章：

- Nuxt3混合渲染实践：SSR、SSG与CSR如何取舍
- FastAPI + pgvector实现站内RAG问答
- 从零实现前端监控SDK
- Web Vitals性能优化记录

2. 知识专题

按专题组织内容，而不是只按时间归档。

专题建议：

- 前端工程化
- Vue / Nuxt
- AI / RAG
- 性能优化
- 项目复盘

3. 智能问答入口

展示一个轻量提问框：

```txt
从站内知识库提问...
```

点击后跳转 `/ask`，或展开全局悬浮问答。

4. 系统能力指标

展示真实或开发环境数据：

- 已发布文章数
- 知识库分片数
- 最近一次向量同步时间
- 平均问答耗时
- LCP / CLS / INP状态
- 最近7天JS错误数

这些指标能让项目看起来像真实产品，而不是静态页面。

# 五、文章体验设计

文章详情页是最重要的展示页，应重点打磨。

## 5.1 页面结构

```txt
顶部导航

  - 标题
  - 摘要
  - 标签
  - 发布时间
  - 阅读时间
  - 作者小头像
正文区域
  - Markdown正文
  - 代码高亮
  - 图片/流程图
右侧目录
底部
  - 相关文章
  - 基于本文提问
  - 评论区
```

## 5.2 文章页能力

必须实现：

- Markdown渲染；
- 代码高亮；
- 复制代码按钮；
- 文章目录；
- 阅读进度条；
- 图片懒加载；
- 相关文章；
- 标签跳转；
- SEO title / description；
- “基于本文提问”按钮。

可选增强：

- 暗色阅读模式；
- 字号切换；
- 文章收藏；
- 文章内锚点分享。

## 5.3 图片策略

文章图片不要使用无意义风景图。

推荐图片类型：

- 架构图；
- 流程图；
- 代码截图；
- Lighthouse报告截图；

- RAG检索结果截图；

# 六、RAG问答设计

RAG是项目差异化亮点，不应只做普通聊天框。

## 6.1 两种入口

1. 全局悬浮问答

适合在任意页面快速提问，保持轻量。

2. `/ask` 独立问答页

展示完整问答体验和引用来源，是面试演示重点页面。

## 6.2 问答结果结构

每次回答建议展示：

- 用户问题；
- AI回答；
- 引用来源；
- 来源文章标题；
- 来源片段；
- 相关度；
- 点击跳转原文；
- 无命中兜底；
- 失败重试；
- 加载状态。

## 6.3 交互状态

必须覆盖：

- 初始空状态；
- 输入中；
- 请求中；
- 流式输出中；
- 成功；
- 无匹配内容；
- 接口失败；
- 超时；
- 重试。

## 6.4 示例空状态文案

```txt
可以试着问：
- Nuxt3中SSR和SSG如何选择？
- 这个项目如何实现前端监控？
- pgvector在RAG里负责什么？
```

# 七、后台设计

后台不追求复杂，但要像真实运营工具。

## 7.1 Dashboard

展示：

- PV / UV；
- 文章阅读排行；
- RAG提问次数；
- 平均问答耗时；
- LCP / CLS / INP；
- JS错误数；
- 接口错误数；
- 最近向量同步状态。

## 7.2 文章管理

功能：

- 文章列表；
- 标题搜索；
- 分类筛选；
- 标签筛选；
- 发布状态筛选；
- 新建文章；
- 编辑文章；
- 删除文章；
- 手动同步向量；
- 查看向量同步状态。

文章状态：

```txt
draft       草稿
published   已发布
archived    已归档
```

向量状态：

```txt
pending     待同步
synced      已同步
failed      同步失败
```

## 7.3 文章编辑器

编辑器采用 Markdown 编辑，不做复杂富文本。

字段：

- 标题；
- 摘要；
- slug；
- 分类；
- 标签；
- 封面图；
- SEO title；
- SEO description；
- Markdown正文；
- 发布状态。

编辑体验：

- 左侧编辑，右侧预览；
- 草稿自动保存；
- 发布前校验；
- 发布后触发向量同步；
- 同步失败时后台提示。

## 7.4 知识库运维

展示：

- 总文章数；
- 总分片数；
- 已同步文章数；
- 失败文章数；
- 最近同步时间；
- 单篇重建；
- 全量重建；
- 重建进度。

## 7.5 监控大盘

展示：

- Web Vitals趋势；
- 页面访问排行；
- JS错误列表；
- Promise异常列表；
- 接口错误列表；
- 资源加载失败；
- 曝光数据；
- 用户停留时长。

# 八、头像与图片展示

## 8.1 头像

建议放头像，但不要做成传统个人主页大头像。

使用位置：

- 导航栏小头像或Logo；
- About页面正式头像；
- 文章详情作者信息小头像。

建议：

- 可以使用真实头像，专业、清晰即可；
- 如果不想露脸，可以使用个人技术Logo；
- 不建议使用AI生成真人头像。

## 8.2 文章封面

文章封面不是必须，但精选文章建议有小封面。

推荐封面：

- 架构图；
- 代码截图；
- 监控图表；
- 流程图；
- 页面截图；
- 简洁几何图。

不推荐：

- 随机风景图；
- 暗色模糊科技背景；
- 无意义插画；
- 与文章内容无关的素材。

## 8.3 项目截图墙

About页面或首页下方可以放项目截图墙：

- 首页截图；
- 文章详情截图；
- RAG问答截图；
- 后台Dashboard截图；
- 监控大盘截图；
- Lighthouse结果截图。

这部分对简历和面试非常有价值。

# 十一、简历展示重点

前端最终应能在简历和面试里讲清楚：

1. 为什么用Nuxt3，而不是普通Vue SPA；
2. 哪些页面用SSR、哪些用SSG、哪些用CSR；
3. Markdown阅读体验如何做；
4. RAG问答如何展示引用来源；
5. 前端监控SDK采集了哪些指标；
6. 后台如何管理文章和向量同步；
7. Lighthouse / Web Vitals / Playwright结果如何证明项目质量。

---

# 十二、待确认问题

后续需要继续确认：

1. 是否使用真实头像；
2. 是否做暗色模式；
3. 是否首版实现评论；
4. 文章封面图是否强制上传；
